Architecture

Проект состоит из связанных pipeline-маршрутов вокруг stage_id. Каждый stage начинается с изображения, создает анализ и prompts, выполняет генерацию, доставляет результаты и затем участвует в post-production workflow.

Four Core Contracts

  1. GenerationConfig в config.py — единая точка правды для generation-флагов.
  2. Все артефакты одного этапа имеют общий префикс stage_id.
  3. output/ — временная зона; итоговая доставка идет через utils/project_delivery.py.
  4. Sequence-утилиты читают stage-based артефакты из regeneration_assets_dir.

High-Risk Lifecycle

main_full_pipeline.py и main_grok_batch.py могут очищать рабочие папки или удалять обработанные input-файлы. Любое изменение cleanup-поведения требует отдельной проверки.

Data Flows

Full Generation Route

input image→main.py→analysis→prompt synthesis→output/ stage artifacts→Grok runtime→delivery

Grok Runtime Route

  1. main_grok_web.py получает image и *_v_prompt_*.
  2. Если включен generate_source_background, сначала создается background image.
  3. Grok генерирует video.
  4. utils/project_delivery.py копирует media в final_videos_dir.
  5. main_grok_batch.py повторяет это по всем prompt-файлам.

Portrait/Image Batch Route

input images→main_chatgpt_portrait_batch.py→ChatGPTGeminiGrokOpenAI API→output/chatgpt_*output/gemini_*output/grok_*
  • One job is created for each image/style pair.
  • Result filename: <image_stem>_<style_slug>.png.
  • --skip-existing is the restart contract.
  • Gemini and Grok mirror output/chatgpt_* into output/gemini_* and output/grok_* unless --output-dir is explicit.
  • --delivery-config-file config_*.json keeps restart copies in project output/... and also copies each saved PNG into final_output_dir with the same relative subfolder path, such as output/grok_portraits/... → final_output_dir/grok_portraits/....

Invariants

Configuration
  • Every new generation flag must be added to config.py.
  • Update the field set, GenerationConfig, from_dict(), override(), docs, and tests.
  • Only one framing mode may be enabled at a time.
  • ai_optimal_then_identity_safe_ai_optimal_percent is a ratio parameter, not a separate mode.
Artifacts
  • Stage artifacts must share the same stage_id prefix.
  • Naming changes must be checked everywhere filenames are written, read, parsed, synced, or reported.
Lifecycle
  • main_grok_batch.py clears input/ and output/ after successful batch unless --keep-workdirs is set.
  • main_full_pipeline.py can remove processed input images.
  • Cleanup behavior changes are high-risk.
Prompts
  • Prompt semantics must stay aligned across OpenAI synthesizer, local PromptBuilder, manifests, docs, and tests.
  • If EN and RU prompt artifacts are written, semantic changes must not affect only one language branch silently.
Delivery
  • .mp4 files go to final_videos_dir.
  • Portrait/image-edit PNG copies go to final_output_dir when --delivery-config-file is present, preserving the project output/... subfolder structure.
  • Non-video stage assets go to regeneration_assets_dir/<stage_id>/.
  • Background image handling must not break non-video sync.

Change Control

  1. Classify the change: config, prompt, scene schema, naming, Grok runtime, delivery, optimizer, reports, portrait batch.
  2. Open project_structure_registry.json.
  3. Select the matching change_type.
  4. Review all must_touch and must_review files.
  5. Update user-facing docs if behavior changes.
  6. Add or update tests for changed contracts.
  7. Run at least one targeted test.
  8. For high-risk changes, run an integration route to a real artifact.
